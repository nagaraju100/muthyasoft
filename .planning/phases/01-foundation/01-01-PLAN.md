Run create-next-app with non-interactive flags to create the project in the current directory:

```bash
npx create-next-app@latest . --typescript --tailwind --eslint --app --src-dir --turbopack --import-alias "@/*" --yes
```

Flag explanation:
- `.` - Create in current directory (project root already exists)
- `--typescript` - Enable TypeScript
- `--tailwind` - Include Tailwind CSS
- `--eslint` - Include ESLint
- `--app` - Use App Router (not Pages Router)
- `--src-dir` - Use src/ directory structure
- `--turbopack` - Enable Turbopack for faster dev builds
- `--import-alias "@/*"` - Set up @ alias for src/ imports
- `--yes` - Skip confirmation prompts

If the current directory has existing files (like README.md), create-next-app may prompt. In that case, accept overwriting or run in a clean state.

After creation, verify the structure matches research recommendations:
- src/app/ exists with layout.tsx, page.tsx, globals.css

- package.json has next, react, react-dom, typescript dependencies
